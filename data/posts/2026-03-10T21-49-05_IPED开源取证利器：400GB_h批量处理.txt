【标题】
IPED开源取证利器：400GB/h批量处理

======================================================================
【正文】
IPED是一款源自巴西联邦警察的开源取证软件，最初是用Java写的，2012年就开始内部使用，直到2019年才把代码正式推到GitHub上。项目在GitHub Trending上得到292⭐的关注，说明在取证社区里有一定的影响力。它的核心目标就是处理大批量案件时的稳定性和效率，所以从一开始就强调命令行批处理，支持可移动介质直接运行，不需要在目标机器上装 anything。

从技术实现角度看，IPED的代码结构相当直接，使用Maven做构建，需要JDK 11和JavaFX（比如Liberica OpenJDK 11 Full JDK）。只要把JAVA_HOME指向 JDK 安装目录，然后执行git clone后进入目录跑mvn clean install，就会在target/release目录得到一个快照版。这里要注意 master 分支是开发分支，容易不稳定，想要正式版的朋友可以 checkout 相应的 release tag。在Linux下还得单独编译Sleuthkit以及它的依赖，官方文档里有详细的步骤，别忘了按部就班装好。

IPED支持的磁盘映像格式非常广，包括RAW/DD、E01、ISO9660、AFF、VHD、VMDK，还有EX01、VHDX、UDF(ISO)、AD1（AccessData）和UFDR（Cellebrite）这些常见的取证格式。它只依赖Sleuthkit来解析磁盘映像，所以所有这些格式都能被识别。处理速度上，官方在2019年12月的数据显示单机可以做到400GB/h的处理速率，还能一次性处理超过1.35亿个项目项，这对于大案子来说是很香的。

功能上，IPED几乎覆盖了取证里常用的全部环节。支持的哈希算法有md5、sha-1、sha-256、sha-512还有edonkey，警察部门还能用PhotoDNA进行图片识别。Hash集合支持NIST NSRL、NIST CAID、ProjectVIC、Interpol ICSE，还有标准CSV格式。哈希去重是快速的，能够在大量数据里快速定位重复文件。文件签名分析、文件类型分类、属性标签化，还有递归展开容器（比如ZIP、7z）的能力，都是它的强项。磁盘镜像的碎片支持也很全，单个DD、E01、EX01、VHD、VHDX、VMDK都能直接读取，甚至 differential VMDK也能处理。

图片和视频的展示也很人性化，几百种格式的缩略图都能直接在界面里浏览。地理信息支持 GPS 坐标，可以直接在地图上标记，使用 Google Maps、Bing 或 OpenStreetMap 任选其一。正则搜索功能非常强大，能够自定义脚本来匹配信用卡号、邮箱、URL、MAC地址、比特币地址、以太坊钱包等，甚至可以做钱币价值的匹配。内置十六进制查看、Unicode文本、元数据和原生查看器，让你能够深入查看文件的每个细节。

索引和搜索部分，IPED会对文件内容、元数据以及未分配空间进行全文索引，支持模糊搜索和关键字匹配。它的 Carving 引擎非常高效，只占整体处理时间的约10%，但能扫描超过40种文件格式，包括各种视频，并且支持通过脚本扩展。OCR功能使用 Tesseract 5，能够识别图片里的文字，这在处理扫描文档时很有帮助。加密检测通过已知模式和熵测试实现，能够快速标记可疑的加密容器。

处理模式方面，IPED提供了 forensic、pedo（CSAM）、triage、fastmode（预览）和 blind（自动提取）五种配置文件，满足不同场景的需求。语言识别支持70多种语言，Named Entity Recognition需要下载 Stanford CoreNLP 的模型才能使用，但一旦装好就能自动抽取人名、组织机构、地点等信息。过滤规则可以基于任意元数据自定义，类似文件的相似度搜索也可以设定阈值，图像相似搜索支持内部或外部图库，面部识别则不需要 GPU，只是调节阈值即可。

最后要提的是，IPED的源码完全开放，社区贡献非常欢迎。如果你想参与进来，记得先看看 Contributing 指南，遵守代码风格和提交规范。总的来说，IPED之所以在开源圈子里受关注，就是因为它把取证这一步的繁琐和低效给抽象成了命令行和批处理，再配合强大的多线程和多格式支持，让大量案件能够在短时间内得到处理。对于需要处理海量数字证据的取证工程师来说，它是一个值得尝试的利器。

======================================================================
【描述】
IPED 是一款源自巴西联邦警察的开源取证工具，最近在 GitHub Trending 中斩获 292 ⭐，引发社区关注。它采用命令行批处理，支持直接从移动介质运行，无需在目标机器装任何依赖，这在取证场景下是一个突破。

它兼容 RAW、E01、VHD、VMDK 等 12 种磁盘映像格式，单机可达 400 GB/h 处理速率，并提供 md5、sha‑256、edonkey 等多算法哈希、NIST NSRL 白名单以及 PhotoDNA 图像识别。 additionally 支持批量去重、容器展开、文件签名分析和多线程加速。

开发者可直接通过 Maven 编译，在 Linux 上单独编译 Sleuthkit，生成 target/release 快照；适用于大案件的快速审查、取证报告生成、证据链重建等场景，尤其在需要批量处理数百 GB 证据时尤为有用。

亮点在于全链路命令行可编排、多语言识别、脚本化过滤；局限是 Windows 环境下需额外配置 JavaFX，且社区活跃度仍在增长。整体看，它展示了开源取证工具向自动化、可扩展方向的必然趋势。

想亲自体验？克隆仓库后跑 `mvn clean install`，在 GitHub Issue 区分享你的取证案例，让更多人一起提升效率。
