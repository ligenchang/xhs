【标题】
我用 vibe coding 做了一个梦中的 macOS 演示应用 小记

======================================================================
【正文】
大家好， 我是 Lucy, 来自小红书的大模型观测站， 我们专注于挖掘和分享最新、最有用的AI工具和模型， 帮助开发者提升效率， 抓住AI浪潮中的机会。

上周末我在山景城参加了 Social Science FOO Camp， 这个活动是典型的 unconference 形式， 没有人需要提前提交议题， 现场有空槽就可以上台分享。 我抢到了一个时段， 标题是《LLMs 状态 2026 年 2 月版》， 副标题写着"自从 11 月以来一切都变了!". 当时我想给这次演讲加点小把戏， 于是决定在前一天晚上用 vibe coding 快速做一个属于自己的 macOS 演示程序。

我之前已经写过几篇关于过去一年 LLM 发展的回顾， 分别在 2023、2024、2025 年底发过。 去年六月在 AI Engineer World’s Fair 我也做过《最近六个月的 LLMs， 用骑自行车的鹈鹕说明》， 那次是六个月的回顾。 这次我故意把时间窗口缩到只有三个月， 正是为了展示加速的节奏——毕竟 2025 年 11 月确实是个明显的拐点。 为了强调这一点， 我当天还穿了之前拿到的 Gemini 3 毛衣， 结果毛衣才穿两周就已经过时了， 因为 Gemini 3.1 已经出来。

按照我在斯坦福学到的 STAR 原则， 每次演讲我都喜欢放进一个"他们永远会记得"的小把戏。 这次我准备了两个： 第一部分是围绕数据分析的卡卡鹦鹉繁殖季节做例子， 顺便展示了我的那个印有卡卡鹦鹉的马克杯； 第二部分则是快速展示一些骑自行车的鹈鹕图片， 最后揭示整个演示其实都是我在前一天晚上用 vibe coding 写好的 macOS 应用跑出来的。

这个应用叫 Present， 名字真的就是我当时脑子里第一个蹦出来的词。 它是用 Swift 和 SwiftUI 写的， 编译完之后只有 355KB， 压缩后竟然只有 76KB。 我说这句话的时候其实有点紧张， 因为我很少看到这么小的原生 macOS 程序， 不过 Simon Willison 在他的博客里确实提到了这个数字（来源：Simon Willison 博客）。

我平时做演示大部分时候会用 Keynote， 不过有时候我想换种方式， 用一串网页页码来代替幻灯片。 具体做法是打开一个浏览器窗口， 每个标签页放一张幻灯片对应的 URL， 然后讲的时候依次点标签切换。 这种方法挺直观， 但最大的风险是如果浏览器崩溃， 整套幻灯片就全没了。 虽然我会把所有 URL 备份在一个笔记文件里， 出事的时候可以手动重新打开， 但那种在现场慌乱地找文件的感觉真的很糟。

于是我在深夜给自己定下了一个起始需求： 构建一个基于 SwiftUI 的演示程序， 每张幻灯片就是一个 URL。 程序启动时显示一个窗口， 右边是 WebView 用来加载网页， 左边有一个侧边栏可以增删、重排这些 URL。 点击菜单里的 Play（或者快捷键 Cmd+Shift+P）后程序进入全屏， 左右方向键切换网页， 上下键可以调节字体大小或者滚动页面， 按 Esc 退出全屏。

我把这个需求塞进了我当时在用的 vibe coding 助手里， 它很快就给出了一个实现计划。 你可以在 Simon Willison 的博客里看到完整的对话记录（来源：Simon Willison 博客）， 这里我不打算逐行贴代码， 只说一下大致实现。

在 Present 里， 一个演示就是一个有序的 URL 列表。 左侧的 UI 只有三个按钮： 加一个新 URL、删除选中的 URL、以及上下移动来改变顺序。 所有编辑操作都是即时生效的， 而且每次修改后程序会自动把当前的 URL 列表写到磁盘上的一个临时文件里。 这样一来， 即使程序意外崩溃， 重新启动的时候它会读取这个临时文件， 把上次的编辑状态恢复回来。 这点对我来说是个巨大的安心点， 因为我曾经在一次内部 demo 中因为 Xcode 异常退出而现场丢失了整套幻灯片， 那种感觉实在太糟了。

除了自动保存， Present 还支持把演示导出为纯文本文件。 导出的格式其实就是每行一个 URL， 没有其他的标记或者 JSON 包装。 导入的时候只要打开那个文件， 程序会逐行读取并重建 URL 列表。 这个设计源自于我过去常用的办法——把所有链接复制到一个 txt 文件里， 不过现在我终于把这件事做到了自动化。

最初的版本完成后， 我发现自己还有点意犹未尽。 既然已经有了一个能够全屏切换网页的小工具， 为什么不再给它加个遥控功能呢？ 于是我又给助手下了一个新的提示： 在本机的 0.0.0.0:9123 上开一个简易的 HTTP 服务器， 提供一个手机友好的网页， 页面上只有两个巨大的左箭头和右箭头按钮， 点击的时候分别向服务器发送上一张或者下一张幻灯片的指令。

助手几乎立刻就给出了实现思路： 使用 Swift 的 NWListener 或者普通的 Socket 建立 TCP 监听， 每当收到 GET 请求时， 根据查询参数决定是向左还是向右移动当前的 URL 索引， 然后在主线程里更新 WebView 的内容。 为了让手机页面看起来不至于太丑， 我简单写了一个带一点内联 CSS 的 HTML， 放在 Bundle 里随程序一起打包。 就这样， 我在深夜把代码加进去， 编译通过后用 iPhone 连上同一局域网的 Wi-Fi， 打开浏览器输入 http://<电脑IP>:9123， 发现左右按钮真的能够控制演示的前进和后退。

实际使用的时候， 我发现自己在讲台上更倾向于用手机遥控而不是键盘。 一方面是因为左手可以自然地握着麦克风， 右手则可以随意点击手机屏幕； 另一方面是因为有时候我想在讲某个网页的时候稍作停顿， 这时候用手机轻点一下比去键盘上找左键更不容易打断我的思路。 当然， 这个遥控也不是完全没有问题， 比如如果手机和电脑之间的网络出现波动， 指令可能会延迟或者丢失， 不过在我的 FOO Camp 演示期间， 整场基本都是顺畅的， 没有出现明显的卡顿。

演结束后， 我把 Present 的源码放到了自己的 GitHub 仓库里， 许可证采用了 MIT。 我故意没有把它做得太“产品化”， 因为我觉得这个工具更像是一个个人的实验品——它解决了我自己在使用 Keynote 或纯网页幻灯片时遇到的痛点， 但如果要让它变成每个人都能直接下载使用的产品， 还需要加很多比如密码保护、主题切换、导出 PDF 等等功能。 不过正如 Simon Willison 在博客里所说（来源：Simon Willison 博客）， 有时候一个能够在 45 分钟内完成并能在真实场景中使用的小工具， 比一个半年才能发布的完整产品更能体现工程师的即时创造力。

回顾这一次的经历， 我发现 vibe coding 真的不是一个营销噱头， 而是一种能够让我在极短时间里把想法变成可运行代码的实际方式。 当然， 这也需要我对所使用的框架（这里是 SwiftUI 和 Swift）有一定的熟悉度， 否则光凭提示语可能会得到很多编译错误或者运行时崩溃。 我在写 Present 的过程中， 确实遇到过几次因为忘记在主线程更新 UI 而导致的崩溃， 后来我在代码里加上了 DispatchQueue.main.async 包装， 问题就解决了。 这些小插曲其实正是开发过程中很常见的“瑕疵”， 如果把它们全部去掉、只保留一段滔滔不夸的成功叙事， 反而会失去真实感。

现在我想把这个小程序分享给大家， 如果你也经常在演示中需要快速切换网页， 或者你想自己动手做一个极轻量级的演示工具， 可以去看看我的仓库（这里不放具体链接， 你可以搜索 Present Simon Willison 或者直接在 GitHub 上找到我最近的项目）。 当然， 如果你只是想要一个即用即丢的方案， 直接把几个网页链接保存到一个 txt 文件里， 用浏览器的标签页切换也能凑合。 但如果你愿意花一点时间尝试用 SwiftUI 构建自己的小工具， 相信你也会像我一样在那深夜里， 看着只有几百 KB 的编译产物， 感到一种难以言说的满足。

最后， 我想说的是， 演讲工具不过是手段， 真正让人记得的还是你在这些幻灯片背后讲述的故事和你对技术的热情。 不管是用 Keynote、网页， 还是自己 vibe coding 出来的 Present， 核心永远是你想要传递的思想。 希望大家都能在接下来的演讲中， 既有靠谱的工具， 也有让人印象深刻的内容。 好了， 今天的分告到这里， 如果你对 Present 有任何想法或者想一起改造它， 欢迎在评论区留言， 我们下次再见。

【描述】
近年来，AI 辅助编程正从概念走向实战，开发者们越来越需要能够在几小时甚至几十分钟内把想法变成可运行原型的工具。这种快速迭代的能力不仅提升了个人效率，也让我们在技术分享、内部演示等场景中拥有了更灵活的表现形式。

Lucy 在参加 Social Science FOO Camp 前一天晚上，仅用 vibe coding 就完成了一个名为 Present 的 macOS 演示程序。该应用基于 SwiftUI 和 Swift，编译后仅 355KB，压缩后只有 76KB；左侧侧边栏管理 URL 列表，右侧 WebView 加载网页，支持全屏播放、左右方向键切换、上下键调节字体或滚动，并且每次编辑都会自动写入临时文件，防止意外崩溃导致数据丢失。她随后为该程序添加了局域网 HTTP 遥控功能，用手机浏览器打开电脑 IP:9123 即可通过巨大的左右箭头按钮远程控制幻灯片前进与后退。

对于经常需要在演示中切换网页或展示在线内容的开发者来说，Present 提供了一种极轻量级的替代方案。无需依赖 Keynote 或复杂的网页标签页管理，只需把想展示的链接保存为纯文本文件，程序即可快速导入并播放。其自动保存和断点恢复机制让现场演示更加安心；而手机遥控则解锁了左手持麦克风、右手点击屏幕的自然操作姿势，尤其适合需要即时互动或现场演代码的场景。MIT 许可证也意味着你可以自由 fork、修改，甚至将其集成到自己的工作流中。

虽然 Present 的实现令人惊艳，但它依然是一个个人实验品：功能聚焦在 URL 列表与网页切换，缺少主题切换、密码保护、导出 PDF 等产品级特性；遥控依赖局域网稳定性，网络波动时可能出现指令延迟或丢失。不过正是这种“有所取舍”的设计，让它在 45 分钟内从零到可用，完美体现了 vibe coding 的即时创造力。正如 Simon Willison 所言，能够在真实场景中使用的小工具，往往比半年后才发布的完整产品更能展示工程师的解决问题能力。
