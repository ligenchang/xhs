Claude Code Max费用：500≈5000

【正文】
嘿，最近刷子上看到不少关于 Claude Code Max 费用的争论，有的说 Anthropic 每月要收 5000 美元，有的说其实只要几百块。我个人觉得这种说法有点儿过头，下面把我自己琢磨的过程写出来，顺便给大家点儿实情。

首先得说说那篇 Forbes 的文章，里面提到 Cursor 认为 Anthropic 的 200 美元月费套餐在实际使用中可以燃掉约 5000 美元的算力。这个数字听起来很吓人，但背后的算式其实是基于 Anthropic 官方的 API 价格——输入 5 美元/百万 token，输出 25 美元/百万 token。换句话说，如果一个人把自己一天都塞满 token，理论上确实能把月账单推到 5000 美元左右。

不过这里有个关键点，API 价格和 Anthropic 真实的算力成本可不是一回事。我们可以把它跟 OpenRouter 上的开源模型价格对比一下。比如 Qwen 3.5 397B-A17B 在 OpenRouter 上的输入价格才 0.39 美元/百万 token，输出 2.34 美元/百万 token。再比如 Kimi K2.5 1T 参数模型，输入 0.45，输出 2.25。相对来看，Anthropic 的价格大概贵了 10 倍左右。而且这些开源模型的提供者还得付出 GPU 费用、维护服务器、还要赚点钱，所以他们不可能在亏本的情况下一直卖这么便宜。

如果把 Claude Code Max 的用户在 Anthropic 那里消耗的 5000 美元 API 费用折算成实际算力成本，大约只需要 500 美元左右。也就是说，最激进的用户，Anthropic 实际亏的可能是 300 美元左右，而不是 4800 美元。但这只是一种理想情况，真实的使用情况要复杂得多。

Anthropic 在推出周额度的时候也提过，只有不到 5% 的订阅者会触碰到上限。我自己用 Max 20x 方案，一周大概消耗一半的 token 预算，感觉已经很难再加速了，除非真的想把自己逼出 RSI。〝重度使用〟

更值得玩味的是，真正的 5000 美元成本其实来自 Cursor 自己。它们需要支付 Anthropic 官方的 API 价格来获得 Opus 4.6 的访问权限。于是，如果 Cursor 想在自己的平台上提供类似 Claude Code 的体验，单用户每月大概要花 5000 美元。但对 Anthropic 来说，这背后的真实成本可能只有 500 美元。换句话说，Cursor 在算钱，而 Anthropic 并不一定在亏钱。

当然，Anthropic 并不是一家盈利的公司，但推测的原因并不全是因为推理成本。还有其他因素，比如品牌效应、开发者粘性以及模型在某些任务上的优势。相对来说，开源模型虽然便宜，但在某些场景下仍然不如 Anthropic 的模型表现得那么好——至少在现在的时间点是这样。

总的来说，这篇文章的核心信息是：$5k 并不是 Anthropic 实际的支出，而是 Cursor 为了使用官方 API 所需的费用。Anthropic 真正的成本可能只有 $500，甚至可能在盈亏平衡点上。开发者们在选型时，应该更关注实际使用的费用结构，而不是被媒体头条的夸张数字误导。

Cache 的读取费用差距也很大，Anthropic 对缓存读取要收 0.5 美元/百万 token，而 OpenRouter 只收 0.07 美元，差了七倍。这意味着，如果你的应用里有大量重复请求，缓存能帮你省下不少钱。不过 Anthropic 的缓存机制在实现上可能比开源的更复杂，有时候甚至会因为模型的更新而失效，导致实际节省不如预期。

另外，推理的延迟也和成本有直接关系。GPU 资源有限，如果一个请求需要更长的上下文，服务器需要保持更多的并发实例，这会进一步推高费用。于是，很多开发者在设计提示词时会刻意控制 token 数量，尽量把最重要的内容压缩进去，而不是把整段文档全塞进去。

从商业角度看，Anthropic 之所以愿意保持高价，除了技术优势，还因为他们的模型在代码生成、多步推理等方面确实表现得更好。很多开发者在实际项目中发现，使用 Claude Code 能够把调试时间缩短一半，甚至在某些复杂算法实现上比开源模型快得多。这种效果带来的收益往往可以抵消部分的费用。

不过，也有不少开发者选择用开源模型来降低成本，尤其是在资源有限的个人项目或者小团队里。他们会把模型部署在自己的服务器上，或者使用一些便宜的云服务，自己承担算力成本。这样一来，虽然单次调用的费用低得多，但维护成本、更新维护的时间成本也会随之增加。

核心要点其实只有两点：一是 Anthropic 官方的输入/输出 token 价格在业内算是偏高，约为 5 美元/百万 token 输入、25 美元/百万 token 输出；二是实际消耗的算力成本远低于表面的 5000 美元，估算后仅在几百美元左右。换言之，媒体抢眼的数字更多是营销夸张，而真实的费用结构要更为细腻。

从实用角度看，Claude Code Max 为高强度代码生成、多步推理提供了强大的加速能力，尤其在需要频繁调试或实现复杂算法的场景下，能够把原本数小时的工作压缩到几分钟。如果你是一名需要快速原型验证的创业者，或是团队里负责核心业务逻辑的工程师，试用这个工具可以显著提升产出效率。
